# Builders Atlas AI Engineering Master Specification

## Objective
Build a production-ready Builders Capture platform that becomes the trusted data collection layer for Builders Atlas.

## Source of Truth
Read and follow every document in the `.ai` directory before writing code.

## Development Order
1. Infrastructure
2. Authentication
3. Database
4. Mission Engine
5. Capture Flow
6. AI Extraction
7. Verification
8. Offline Sync
9. Profile
10. Production Deployment

## Engineering Rules
- Mobile First
- Offline First
- TypeScript Strict Mode
- Feature-based Architecture
- Reusable Components
- Test Coverage for Critical Flows
- No Mock Data in Production Code
- Human Verification Required Before Publishing Data

## Definition of Done
- End-to-end mission flow works.
- Observations persist correctly.
- AI suggestions are reviewable.
- Offline sync is reliable.
- Security and RLS are enforced.
- Application is deployable to production.
